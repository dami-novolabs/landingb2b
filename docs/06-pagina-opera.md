### Decisión sobre el ongoing

- En Opera, **el ongoing es relevante** porque la infraestructura necesita mantenimiento
- En Innova, el ongoing es opcional (delivery de construcción, no mantenimiento)
- Por eso, esta página menciona el ongoing en la card del programa, no como sección aparte

## 10 — FAQs específicas de Opera

### Lista de preguntas

```
1.  ¿Quién es dueño de la infraestructura que construyen?
2.  ¿Cómo manejan la seguridad y los datos sensibles?
3.  ¿Pueden trabajar sobre nuestro CRM/ERP existente?
4.  ¿Qué pasa si necesitamos cambiar algo después?
5.  ¿Cuánto tiempo necesita dedicar nuestro equipo?
6.  ¿Cómo miden el ROI real al cierre?
```

### Respuestas

```
1. ¿Quién es dueño de la infraestructura que construyen?

   Tu empresa. Las cuentas, las API keys, los workflows, los prompts,
   la documentación — todo queda a nombre de tu empresa. Si decidieran
   cortar el contrato con Novo Labs al cierre del programa, no perdés
   nada. Esa es la regla.

2. ¿Cómo manejan la seguridad y los datos sensibles?

   En la fase 1, hacemos un mapeo explícito de qué información sale y
   qué información se queda dentro de tu infraestructura. Para datos
   sensibles trabajamos con modelos que no entrenan con tus datos
   (Claude Enterprise, Azure OpenAI), encriptación en reposo y en
   tránsito, y access control a nivel de proceso. Cada decisión queda
   documentada y aprobada por la dirección.

3. ¿Pueden trabajar sobre nuestro CRM/ERP existente?

   Sí. Trabajamos sobre integraciones (APIs, webhooks, conectores
   nativos) — no reemplazamos tu sistema core. Si tu CRM/ERP no tiene
   API moderna, lo manejamos con automatización de UI (RPA) y lo
   documentamos como límite del scope.

4. ¿Qué pasa si necesitamos cambiar algo después?

   Dos caminos. Si tu equipo técnico participó en la modalidad DWY,
   tienen la capacidad de hacer cambios menores sin necesidad de
   volver a contratarnos. Para cambios mayores o nuevas
   automatizaciones, el mantenimiento ongoing o un nuevo engagement
   son las dos opciones.

5. ¿Cuánto tiempo necesita dedicar nuestro equipo?

   En DFY, mínimo: 1 sesión de 90 minutos por semana con el equipo
   responsable + acceso a los referentes de los procesos durante el
   diagnóstico. En DWY, sumar 8–10 horas por semana de un perfil
   técnico que participe en la construcción.

6. ¿Cómo miden el ROI real al cierre?

   En la fase 1 establecemos la línea base: cuánto tiempo y costo
   consume el proceso hoy. Durante la fase 3, medimos el mismo
   proceso con el sistema construido. La sesión de cierre compara
   métrica antes vs. después con datos reales, no proyecciones. Esa
   métrica es el criterio de éxito firmado al inicio del programa.
```

## Diferencias clave de tono Opera vs. Innova

Para que cada página se sienta como un producto distinto pero coherente con la marca:

| | Innova | Opera |
|---|---|---|
| Verbo del headline | "Lanzá" | "Escalá" |
| Métrica de éxito | "Compromiso de pago" | "Horas liberadas" |
| Sustantivo central | "Oferta" | "Infraestructura" |
| Lenguaje | Producto, mercado, segmento | Procesos, capacidad, ROI |
| Buyer mental | Crecimiento | Eficiencia |
| Imagen mental | Lanzar al océano | Aceitar la máquina |
| Persona técnica que se involucra | CPO, VP Innovación | CTO, COO |

---

## Cierre

Las dos páginas (Innova y Opera) tienen la **misma estructura** intencionalmente. Esto comunica:

- Una firma, dos productos
- El método se aplica con disciplina similar
- El cliente puede compararlas con criterios consistentes

Pero el **contenido específico es diferente** suficientemente para que cada página se sienta como dirigida a un buyer distinto. Esa es la mezcla correcta: consistencia de estructura + variación de contenido.
