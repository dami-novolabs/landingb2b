# Página Track Innova — Wireframe + Copy

> Página: `novolabs.co/innova`
> Trabajo: cerrar al CEO/VP que **ya decidió** que su problema es lanzar algo nuevo con evidencia. Esta página no compite por atención con Opera. Cuando alguien llega acá, ya hizo la elección. El trabajo es solo confirmar y agendar.

---

## Estructura completa (11 secciones)

```
01 — Header (sticky con item "Innova" activo)
02 — Hero específico de Innova
03 — A quién le sirve este programa
04 — Por qué la mayoría de los lanzamientos fracasan
05 — Cómo trabajamos (las 3 fases aplicadas al lanzamiento)
06 — Qué tenés al cierre (entregables específicos)
07 — Lo que NO incluye (límite explícito)
08 — Inversión + estructura de pago + garantía
09 — Caso de éxito (cuando exista; placeholder)
10 — FAQs específicas
11 — CTA final con agendamiento
```

Total: ~5 scrolls. Lectura completa: 2 minutos.

- **"Compromiso de pago de clientes reales"** define qué quiere decir "resultado" en Innova específicamente.

### Por qué este copy

- Los síntomas son **específicos del contexto corporativo**: "presentar al directorio", "directorio", "los últimos 18 meses". Lenguaje del buyer.

## 05 — Cómo trabajamos (3 fases adaptadas a Innova)

### Wireframe

```
┌────────────────────────────────────────────────────────────────────────┐
│                                                                        │
│  CÓMO TRABAJAMOS                                                       │
│                                                                        │
│  12 semanas. 3 fases.                                                  │
│                                                                        │
│  ──────────────────────────────────────────────────────────────────    │
│                                                                        │
│  FASE 1 · SEMANAS 1–4                                                  │
│                                                                        │
│  Diagnóstico y modelo                                                  │
│                                                                        │
│  Antes de validar nada, hay que alinear qué se está intentando         │
│  validar y para quién. Esta fase termina con un criterio de éxito      │
│  firmado por la dirección y un Lean Canvas de la iniciativa.           │
│                                                                        │
│  Qué hacemos:                                                          │
│  · Diagnóstico de cultura de validación en el equipo                   │
│  · Lean Canvas de la iniciativa específica                             │
│  · Definición del beachhead: el segmento más pequeño donde el          │
│    problema es más urgente                                             │
│  · Test DVF (Deseable, Viable, Factible) y prueba de rentabilidad      │
│  · Firma del criterio de éxito                                         │
│                                                                        │
│  Entregable de la fase: Lean Canvas validado + ICP + Pitch             │
│                                                                        │
│  ──────────────────────────────────────────────────────────────────    │
│                                                                        │
│  FASE 2 · SEMANAS 5–8                                                  │
│                                                                        │
│  Descubrimiento del problema                                           │
│                                                                        │
│  La etapa más importante y más omitida. Tu equipo sale a hablar        │
│  con clientes reales, no con sus intuiciones.                          │
│                                                                        │
│  Qué hacemos:                                                          │
│  · Guía de entrevistas con Mom Test (sin contaminar con soluciones)    │
│  · 15–20 entrevistas con clientes del segmento objetivo                │
│  · Análisis individual de cada entrevista (subagente IA + equipo)      │
│  · Síntesis de patrones: qué dicen todos, qué dicen algunos, ruido     │
│  · JTBD priorizado y lenguaje literal del cliente                      │
│                                                                        │
│  Entregable de la fase: Mapa de segmentos + JTBD validado + síntesis   │
│                                                                        │
│  ──────────────────────────────────────────────────────────────────    │
│                                                                        │
│  FASE 3 · SEMANAS 9–12                                                 │
│                                                                        │
│  Solución, oferta y experimento de demanda                             │
│                                                                        │
│  Con el problema validado, diseñar la solución mínima y testear        │
│  si alguien paga. La métrica no es interés — es compromiso.            │
│                                                                        │
│  Qué hacemos:                                                          │
│  · Diseño de solución con criterio 10x y Modelo Kano                   │
│  · Construcción de la oferta usando Ecuación de Valor (Hormozi)        │
│  · Experimento de demanda real: lead magnet, lista de espera,          │
│    carta de intención o preventa                                       │
│  · Decisión basada en evidencia: ir, pivotar o salir                   │
│                                                                        │
│  Entregable de la fase: Propuesta de valor + Oferta + Resultados del   │
│  experimento + Recomendación firmada                                   │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

(Igual al wireframe arriba — es el copy definitivo.)

- Container narrow (max 880px), alineado izquierda
- Cada fase con: eyebrow mono ("FASE X · SEMANAS A–B") / título Display S serif / body M
- "Qué hacemos" como lista con bullet en mono (·)

```

3.  ¿Pueden trabajar con una iniciativa que ya está en desarrollo?
4.  ¿Sirve para B2B o solo para B2C?
5.  ¿Cómo eligen el segmento beachhead?
6.  ¿Qué pasa si el desafío que elegimos al inicio era el equivocado?
```

### Respuestas

```
1. ¿Qué pasa si en el experimento de demanda nadie compra?

   Es uno de los dos resultados posibles. Si la evidencia muestra que
   el segmento no compra, lo sabés con datos en la semana 12 — antes
   de invertir presupuesto en construir. Eso es exactamente el
   resultado que el programa promete: decisiones con evidencia. Un
   "no" temprano es el output más caro que existe en innovación.

2. ¿Cuánto tiempo necesita dedicar nuestro equipo al programa?

   90 minutos por semana de sesión grupal con los fundadores +
   aproximadamente 4 horas de trabajo asíncrono por persona. En la
   fase 2 (entrevistas con clientes), el tiempo sube a 6–8 horas
   por semana de personas designadas. No requiere pausar el negocio.

3. ¿Pueden trabajar con una iniciativa que ya está en desarrollo?

   Sí, siempre que estén dispuestos a re-validar lo que ya construyeron.
   Si el equipo está convencido de que su solución es la correcta y solo
   quiere ayuda para venderla, este no es el programa. Si están abiertos
   a descubrir que el problema o el segmento podrían ser otros, sí
   tiene sentido.

4. ¿Sirve para B2B o solo para B2C?

   Los dos. La metodología es la misma: identificar el problema urgente,
   diseñar la oferta, validar con evidencia. La diferencia está en cómo
   se ejecutan las entrevistas y los experimentos de demanda. Lo
   adaptamos según el caso.

5. ¿Cómo eligen el segmento beachhead?

   En la fase 1. Trabajamos con la dirección para identificar el
   segmento más pequeño donde el problema es más urgente. La regla:
   100 clientes que sufren mucho > 10.000 clientes que sufren un poco.
   El beachhead es donde empieza la validación, no donde termina el
   negocio.

6. ¿Qué pasa si el desafío que elegimos al inicio era el equivocado?

   Para eso existe la fase 1. Las primeras 4 semanas son específicamente
   para evitar ese riesgo: definir el desafío con precisión, alinear
   stakeholders y verificar que tiene las condiciones para resolverse en
   12 semanas. Si en la fase 2 aparece evidencia de que el desafío
   original no era el correcto, lo ajustamos. Preferimos ajustar en la
   semana 5 que descubrirlo en la semana 11.
```

### Layout

- Container narrow, max 720px
- Sin acordeón. Las 6 preguntas + respuestas como bloques continuos
- Pregunta en Display S 32px serif
- Respuesta en Body M 17px Inter, max 60ch
